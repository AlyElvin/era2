# ERA2 — Очередь генераций

Тестовое задание: экран **«Очередь генераций»** для агрегатора нейросетей ERA2.
Список задач генерации (text / image / video / audio), который «живёт во времени» —
задачи сами продвигаются по статусам через клиентский мок-движок (бэкенда нет),
плюс **глобальный плавающий статус-бар** активных генераций поверх любого экрана.

---

## Стек

- **React 19** + **TypeScript** (strict)
- **Vite 7** (сборка/dev), алиас `@/` → `src/`
- **Tailwind CSS v4** + дизайн-токены (semantic + `era` raw) в `src/app/styles/styles.css`
- **State:** React Context + `useReducer` (без Redux — его нет в проекте; структуру предписывает ТЗ)
- **UI-kit:** shadcn-примитивы (`shared/ui`) + собственные `era`-компоненты (`shared/ui/era`)
- **Иконки:** lucide-react · **Анимации:** framer-motion · **Тосты:** sonner · **Валидация:** Zod
- **Шрифт:** Geist / Geist Mono (фолбэк → Inter / системный)
- Менеджер пакетов: **Yarn 4 (Plug'n'Play)**

---

## Запуск

Все команды — из папки `frontend/`.

```bash
yarn install      # установка зависимостей
yarn dev          # дев-сервер → http://localhost:8080
yarn build        # прод-сборка
yarn preview      # просмотр прод-сборки
yarn lint         # ESLint
yarn tsc --noEmit # проверка типов
```

Экран очереди: **http://localhost:8080/queue** (ссылка «Очередь» есть в сайдбаре).

## Архитектура (Feature-Sliced Design)

```
app → pages → widgets → features → entities → shared
```

Импорт между слайсами — только через публичный `index.ts` (без deep-import).

Файлы, относящиеся к заданию:

```
src/
├─ entities/generation-task/      # доменные типы задачи + стартовый сид (8–12 задач)
├─ features/generation-queue/     # вся логика + UI очереди (см. README внутри)
│  ├─ model/                      # reducer, движок, provider, selectors, useQueue, persist
│  ├─ ui/                         # строки/карточки, тулбар, счётчики, состояния, статус-бар
│  └─ lib/                        # форматтеры времени/кредитов
├─ widgets/generation-queue/      # композиция экрана
├─ pages/QueuePage.tsx            # тонкая страница /queue
├─ app/                           # провайдеры, роутер, глобальные стили
└─ shared/                        # ui-kit, lib, hooks, кастомный роутер
```

Единый `QueueProvider` смонтирован на уровне `app` → и страница `/queue`, и глобальный
статус-бар читают **один стор** (счётчики и прогресс всегда совпадают).

Подробная документация фичи (модель состояния, автомат статусов, правила движка,
персистентность, a11y) — [`src/features/generation-queue/README.md`](./src/features/generation-queue/README.md).

## Ключевые решения

- **Мок-движок:** `MAX_CONCURRENT = 2`, FIFO по `createdAt`, тики прогресса ~400–700 мс,
  ~15% сбоев **на задачу** (а не на тик), разная скорость по типу (video/audio медленнее),
  корректная чистка таймеров, без «дотиков» после `cancel`. Изменения только через `dispatch`.
- **Персистентность:** `localStorage` (ключ `era2:generation-queue`), throttled-запись,
  Zod-валидация при чтении. При восстановлении `running` → `queued` (без «зависших» задач).
- **Роутинг:** кастомный лёгкий роутер `@/shared/routing` поверх History API; маршрут `/queue`.
- **Адаптив:** desktop (≥1024) — строки списка; mobile — карточки в стек, статистика 2×2,
  чипы со скроллом, полноширинная нижняя панель статус-бара с учётом safe-area.

## Реализованные бонусы

- **Undo** (sonner) на «Удалить» и «Очистить готовые».
- **Анимации** появления/удаления (framer-motion) + уважение к `prefers-reduced-motion`.
- **Доступность:** `role="progressbar"`, `aria-pressed` на фильтрах, `aria-label` у
  иконочных кнопок, `aria-live` у счётчика активных, видимый focus-ring, клавиатура.

**Вне рамок** (сознательно): юнит-тесты, фильтр по типу, отдельная светлая тема,
виртуализация списка, drag-to-reorder.
